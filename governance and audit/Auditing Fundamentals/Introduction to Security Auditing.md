## What Is Security Auditing?

A **systematic evaluation** of an organization’s security controls to:
- Verify effectiveness  
- Ensure compliance  
- Identify vulnerabilities  
- Support continuous improvement  

---
## Why It Matters

| Purpose | Description |
|--------|------------|
| **1. Find Vulnerabilities** | Uncovers weaknesses before attackers exploit them |
| **2. Ensure Compliance** | Validates adherence to **GDPR, HIPAA, PCI DSS, ISO 27001** |
| **3. Improve Risk Management** | Prioritizes risks by impact/likelihood |
| **4. Strengthen Policies** | Reviews & updates security policies/procedures |
| **5. Support Business Goals** | Protects operations, builds customer trust |
| **6. Enable Continuous Improvement** | Not a one-time task — part of ongoing security hygiene |

---
## Essential Terminology

| Term | Definition | Importance |
|------|-----------|-----------|
| **Security Policies** | Formal rules for protecting info assets | Foundation for all security controls |
| **Compliance** | Adherence to laws/standards (e.g., PCI DSS) | Avoids fines, legal issues |
| **Vulnerability** | Weakness that can be exploited | Primary target of audits |
| **Control** | Safeguard to reduce risk (prevent/detect/respond) | Mitigates vulnerabilities |
| **Risk Assessment** | Process to identify & evaluate risks | Drives security investment priorities |
| **Audit Trail** | Log of system activities/events | Enables forensic analysis & accountability |
| **Compliance Audit** | Check against regulatory requirements | Proves legal/standard adherence |
| **Access Control** | Rules governing who can access what | Prevents unauthorized data access |
| **Audit Report** | Document with findings & recommendations | Actionable output for management |

> ✅ **Security auditing = proactive defense**, not just checkbox compliance.
